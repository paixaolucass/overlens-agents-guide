Parte 01: Fundamentos

O que é um agente de IA, de verdade

Não é só um chatbot com instrução. Agente de IA é um sistema com objetivo, autonomia para agir, memória de contexto e capacidade de usar ferramentas. Entender essa diferença é o primeiro passo para sair do uso superficial.

chat
Uso comum

Prompt → Resposta

Você digita. A IA responde. Conversa encerra. Nenhuma ação no mundo, nenhuma memória real, nenhum encadeamento. Útil para tarefas rápidas, mas o teto é baixo.

  • — Reativo, não proativo
  • — Sem memória entre sessões
  • — Depende de você para cada micro-decisão
  • — Não acessa ferramentas externas
smart_toy
Agente de IA

Objetivo → Execução autônoma

Você define o objetivo. O agente planeja, executa, corrige, usa ferramentas e entrega resultado, com mínima intervenção. Você é o diretor criativo, não o operador.

  • + Autônomo: executa sequências completas
  • + Contextual: mantém identidade e instruções
  • + Ferramental: busca, escreve, calcula, integra
  • + Encadeável: conecta a outros agentes

Os 3 níveis de quem usa IA

Nível 1: A maioria

Delegação passiva

Envia prompt aberto, aceita o primeiro output, não questiona, não itera. Usa a IA como um Google melhorado. Resultado: produtividade marginal.

Nível 2: Usuário avançado

Colaboração ativa

Escreve prompts estruturados, itera com contexto rico, valida outputs, guia a IA com perguntas estratégicas. Resultado: ganhos reais de qualidade.

Nível 3: Construtor

Orquestração de agentes

Constrói sistemas com objetivo, memória, ferramentas e encadeamento. A IA trabalha enquanto você direciona. Resultado: escala e autonomia real.

Parte 02: Tipos

Os 4 tipos de agentes que existem

Cada tipo resolve um conjunto diferente de problemas. Entender para que serve cada um evita o erro mais comum: tentar usar um agente de produção para fazer análise estratégica.

Agente de Pesquisa

Coleta informação, analisa fontes, cruza dados e organiza conhecimento. Substitui horas de pesquisa manual. Ideal para briefings, análise de concorrência, tendências e benchmarking.

Parte 03: Ferramentas

Qual ferramenta usar, e quando

Existem três níveis de complexidade. Os dois principais para criar agentes são o Claude no nível 1, onde você não precisa de código, e o Claude Code no nível 3, onde você tem controle total via terminal.

Nível 1 · Principal
Claude
claude.ai · Projects · claude.com/code

A forma mais direta de criar agentes sem código. Você escreve o system prompt no campo de instrução do Project, sobe os arquivos de contexto, e o agente está funcionando em minutos. Ideal para começar, validar a lógica e usar em produção leve sem infraestrutura.

  • System prompt + arquivos de contexto nativos
  • Sem setup técnico, funciona no navegador
  • Projects com memória de sessão persistente
  • Compartilhe agentes com o time via Project
Nível 1 · API
Claude API
Anthropic SDK · REST API

Acesso direto ao modelo via API. Ideal para integrar Claude em aplicações existentes com controle total sobre mensagens, temperature e parâmetros. Requer código mínimo para começar.

  • SDK oficial em Python e TypeScript
  • Controle de parâmetros e system prompt via código
  • Suporte a streaming e tool calling nativo
Nível 2 · Automação
n8n · Make · Zapier AI
Low-code · Fluxos visuais

Conecta agentes Claude a outros sistemas com gatilhos e condições visuais. Integra com formulários, e-mail, planilhas, Notion, WhatsApp sem escrever código do zero.

Nível 2 · Visual
Flowise
No-code · Interface drag-and-drop

Plataforma visual para construir fluxos de agentes com LLMs. Cria pipelines de RAG, memória e tool calling arrastando blocos. Ideal para prototipar agentes com lógica complexa sem código.

Nível 3 · Principal
Claude Code
Terminal · VS Code · JetBrains

A interface de linha de comando da Anthropic. Instale via npm i -g @anthropic-ai/claude-code e use Claude diretamente no terminal. Lê o código, entende o contexto e age como engenheiro sênior ao lado.

  • Acessa e edita arquivos do projeto diretamente
  • Tool calling, streaming e memória via API
  • Cria, testa e itera pipelines multi-agente
  • Extensões oficiais para VS Code e JetBrains
Nível 3 · Framework
LangChain
Python · TypeScript · Open source

Framework open source para construir pipelines de LLMs com memória, RAG, ferramentas e encadeamento. Extensa biblioteca de integrações com bases de dados vetoriais e APIs externas.

  • Suporte nativo a Claude, GPT, Gemini e outros
  • Chains, agents e retrievers pré-construídos
  • Integração com Pinecone, Chroma, FAISS
Por onde começar

Comece no Claude Projects (Nível 1). Escreva um system prompt completo, suba seus documentos de contexto e use em situações reais por pelo menos duas semanas. Quando sentir o limite da interface, vá direto para o Claude Code (Nível 3): instale no terminal, abra seu projeto e comece a construir pipelines reais. O n8n fica para quando você precisar conectar o agente a sistemas externos sem código. Nunca pule o Nível 1; é lá que você entende o que o agente precisa saber antes de automatizar qualquer coisa.

Parte 04: Tutorial

Como criar seu primeiro agente, do zero

Oito etapas concretas. Sem pular. Cada uma constrói a base da próxima. Se você quiser um agente que realmente funciona, não atalhe o processo.

1
Defina o problema com precisão cirúrgica ›

A maioria cria agentes vagos porque define objetivos vagos. "Um agente para me ajudar com marketing" não é um objetivo; é um desejo. Um objetivo de agente tem que responder a três perguntas com exatidão:

As 3 perguntas obrigatórias antes de criar

1. Qual é o problema específico que ele resolve?
Não "me ajudar com redes sociais". Sim: "Eu passo 4h/semana escrevendo captions para Instagram e o resultado é inconsistente."

2. O que entra e o que sai?
Defina o input (o que você vai fornecer) e o output esperado (o que o agente precisa entregar) antes de escrever uma linha de prompt.

3. Qual é o critério de sucesso?
Como você sabe que o agente funcionou? Sem critério, todo output parece igualmente bom ou igualmente ruim.

// Exemplo: do vago para o preciso // ❌ Vago — vai gerar um agente inútil: "Quero um agente para me ajudar com conteúdo" // ✅ Preciso — vai gerar um agente útil: Problema: "Passo 3h/semana pesquisando referências visuais para briefings de branding" Input: "Briefing com: nome da marca, setor, público, 3 palavras-chave de personalidade" Output: "Documento com 12 referências visuais categorizadas: Identidade, Tipografia, Cor, Layout" Sucesso: "Consigo usar as referências direto no Figma sem nenhuma busca adicional"
2
Mapeie o contexto: quem é esse agente ›

Antes de escrever o prompt, você precisa saber com clareza quem é esse agente: qual é o papel, qual é o tom, o que ele sabe, o que ele não sabe, e como ele se comporta quando recebe inputs ambíguos.

Pense como uma ficha de contratação de um colaborador real. Se você não sabe responder "o que esse agente faz quando recebe um briefing confuso?": você ainda não definiu o agente.

Template de mapeamento de contexto

Papel: [Qual é a função exata deste agente?]
Especialidade: [Em que área ele é especialista?]
Tom: [Como ele se comunica? Direto, analítico, criativo?]
Restrições: [O que ele nunca deve fazer?]
Comportamento em ambiguidade: [O que ele faz quando o input é incompleto?]
Formato padrão: [Como é o output esperado por padrão?]

3
Escreva o System Prompt: a identidade permanente ›

O System Prompt é a constituição do agente. Ele define quem o agente é antes de qualquer conversa começar. Ao contrário das mensagens normais, o system prompt persiste em todas as interações e estabelece o frame de toda resposta.

Um system prompt profissional tem 6 camadas. Cada uma resolve um problema diferente de comportamento:

Identidade Quem é o agente
Define o papel, a especialidade e a perspectiva do agente. Isso ancora todas as outras respostas. Ex: "Você é um especialista em branding com 10 anos de prática em identidades visuais para startups de tecnologia."
Missão Para que o agente existe
Define o objetivo central do agente em uma frase. Sem missão, o agente tenta resolver tudo e não resolve nada bem. Ex: "Sua missão é transformar briefings criativos em documentos de referência visual prontos para uso em projetos de design."
Contexto O que o agente já sabe
Injete o conhecimento base: sobre sua marca, seu público, seu processo, suas preferências. Quanto mais contexto rico, menos você vai precisar repetir nas mensagens. Aqui ficam os arquivos e documentos do projeto.
Regras O que ele deve e não deve fazer
Regras positivas (sempre faça X) e negativas (nunca faça Y). Inclua comportamento em casos ambíguos: o que o agente deve fazer quando recebe um input incompleto ou contraditório.
Tom Como ele comunica
Define a voz: direta, analítica, criativa, técnica. Inclua exemplos de linguagem desejada e indesejada. Tom inconsistente é o sinal mais claro de um prompt mal calibrado.
Formato Como o output chega
Defina a estrutura do output: títulos, seções, listas, tabelas, comprimento máximo. Um formato definido elimina 80% dos ajustes pós-geração. O output deve chegar pronto para uso, não para ser reformatado.
// System Prompt completo — Agente de Pesquisa de Referências Visuais ## IDENTIDADE Você é um curador especialista em referências visuais e estética contemporânea, com domínio em branding, design editorial, motion e direção de arte digital. Pensa como um diretor criativo sênior que já trabalhou com marcas globais e estúdios independentes. ## MISSÃO Transformar briefings criativos em documentos de referência visual densos, organizados e imediatamente utilizáveis por designers e diretores de arte. ## REGRAS DE COMPORTAMENTO - Nunca sugira referências genéricas ou óbvias (nada de Behance top 10) - Sempre justifique cada referência com uma análise de 2 linhas - Se o briefing for vago ou incompleto, faça no máximo 2 perguntas antes de executar - Prefira referências que surpreendam e ampliem o universo estético do cliente - Categorize sempre: Identidade, Tipografia, Cor, Composição, Referência Cultural ## TOM Direto, analítico, confiante. Sem enrolação. Fale como quem domina o assunto. Evite: elogios desnecessários, hedging ("talvez", "pode ser"), linguagem genérica. ## FORMATO DE OUTPUT Título: [Nome do projeto] Universo estético: [2–3 linhas de síntese] [CATEGORIA] - Referência: [Nome / Origem] Análise: [Por que relevante para este briefing] Repita para 12–15 referências distribuídas nas 5 categorias.
4
Defina os inputs: o template de entrada ›

Um agente bom tem uma interface de entrada padronizada. Isso evita que cada vez que você use o agente você tenha que pensar em "o que eu preciso informar". Crie um template de input e use sempre ele.

O template de input serve para três coisas: garantir que o agente receba o mínimo necessário para executar bem, reduzir fricção no uso diário, e tornar o processo ensinável para outras pessoas.

// Template de input — Agente de Referências Visuais BRIEFING DE PROJETO Marca/Projeto: [Nome] Setor/Mercado: [Ex: Fintech B2B, Estúdio de Tatuagem, SaaS Educacional] Público-alvo: [Quem vai ver / usar] Objetivo do projeto: [O que precisa comunicar] Palavras-chave: [3–5 adjetivos que definem a personalidade visual] Anti-referências: [O que você definitivamente NÃO quer] Entregáveis: [Ex: logo, identidade completa, post Instagram, landing page] Referências já usadas: [Para o agente não repetir] // Instrução de uso: Copie esse template. Preencha. Cole numa nova conversa com o agente. O agente sabe o que fazer com isso — está no system prompt.
5
Adicione memória e contexto persistente ›

Memória é o que separa um agente descartável de um agente que melhora com o tempo. Existem três tipos de memória que você pode injetar num agente; cada um serve para uma coisa diferente:

Os 3 tipos de memória de um agente

1. Memória do sistema (via arquivos): documentos que você sobe no Project: guia de marca, portfolio, lista de clientes, preferências pessoais. O agente lê e usa como base permanente.

2. Memória de sessão: o histórico da conversa atual. O agente "lembra" tudo que foi dito desde o início da sessão. Use isso para refinar progressivamente: cada mensagem enriquece o contexto.

3. Memória externa (RAG): para sistemas mais avançados: banco de dados vetorial que o agente consulta dinamicamente. Permite escala ilimitada de contexto. Ferramentas: Pinecone, Supabase Vector, Notion AI.

// O que incluir nos arquivos de contexto do seu Project arquivo: brand_guide.md → Tom de voz, valores, o que a marca não é, exemplos de copy aprovado arquivo: audience_profile.md → Descrição detalhada do público, dores, objeções, linguagem que ressoa arquivo: preferences.md → Referências que você ama, formatos preferidos, regras de estilo pessoal arquivo: past_projects.md → Projetos anteriores que servem como referência de qualidade e estilo // Resultado: o agente já nasce sabendo quem você é. // Você nunca mais precisa se apresentar.
6
Teste com inputs reais, nunca com exemplos fictícios ›

O erro mais caro no desenvolvimento de agentes é testar com inputs inventados. Prompts testados com dados fictícios sempre parecem funcionar bem; o problema aparece só em produção, com dados reais.

Protocolo de teste em 3 rodadas obrigatórias:

Protocolo de 3 rodadas

Rodada 1: Input ideal: Use um briefing real, completo, bem escrito. O agente deveria performar no máximo aqui. Se o output já falhar, o problema está no system prompt.

Rodada 2: Input ambíguo: Use um briefing vago, com informações faltando. Observe como o agente reage: pede para clarificar? Inventa? Trava? O comportamento em ambiguidade revela falhas ocultas.

Rodada 3: Input do pior cliente: Simule o usuário mais descuidado possível. Input curto, sem contexto, fora do escopo. O agente deve se manter coerente ou pedir o que falta, nunca gerar lixo com confiança.

// Checklist de validação de output ✓ O output está no formato definido no system prompt? ✓ As informações são verificáveis (não inventadas)? ✓ O tom está alinhado com a identidade definida? ✓ O resultado está pronto para uso ou precisa de edição pesada? ✓ O agente pediu clarificação quando o input foi vago? ✓ Você conseguiu em < 5min o que levaria 2h manualmente? // Se mais de 2 checks falharam → volte ao system prompt. // Não culpe o modelo. 90% dos problemas estão no prompt.
7
Itere o prompt: como diagnosticar e corrigir ›

Prompt engineering é diagnóstico. Quando o agente entrega algo errado, o problema quase sempre está em uma das seis camadas do system prompt; você precisa saber qual.

Diagnóstico por sintoma

Output genérico / sem personalidade → A camada de Identidade está vaga. Seja mais específico sobre quem é o agente.

Output fora do escopo → A camada de Missão está aberta demais. Restrinja o objetivo com mais precisão.

Tom inconsistente → A camada de Tom não tem exemplos. Adicione "fale como X, não como Y" com exemplos reais.

Formato diferente a cada vez → A camada de Formato não é estruturada. Defina a saída como um template fixo com seções nomeadas.

Comportamento errático em inputs ambíguos → Falta uma regra explícita de comportamento em ambiguidade. Adicione: "Quando o input estiver incompleto, faça exatamente estas 2 perguntas antes de executar."

// Técnica de refinamento iterativo // Em vez de refazer o prompt do zero, use o próprio agente para melhorá-lo: Mensagem para o agente: "Analise o seguinte output que você gerou [cole o output ruim]. Agora analise estas instruções do seu system prompt [cole a seção problemática]. Aponte exatamente onde as instruções são insuficientes para gerar o output correto e sugira como reescrever aquela seção específica." // O agente vai diagnosticar seu próprio prompt. // Use a sugestão como ponto de partida — não como resposta final.
8
Encadeie agentes e integre ao workflow ›

Um agente isolado é poderoso. Uma cadeia de agentes é transformadora. O princípio é simples: o output de um agente vira o input do próximo. Cada agente faz uma coisa muito bem, e passa o bastão.

// Exemplo de pipeline encadeado — Estúdio Criativo Entrada: Formulário de briefing preenchido pelo cliente ↓ Agente 1 — Pesquisa: Recebe o briefing → Gera documento de referências visuais + análise de mercado ↓ Agente 2 — Estratégia: Recebe pesquisa → Gera posicionamento, narrativa central e diretrizes de marca ↓ Agente 3 — Produção: Recebe estratégia → Gera copy para apresentação + textos de interface ↓ Saída: Documento completo de branding pronto para a reunião de apresentação // O que levava 2 dias → 40 minutos // Você revisou + direcionou. Os agentes executaram.
3 formas de integrar ao workflow

Manual (Nível 1): Template de input salvo como snippet. Você cola, o agente executa, você valida. Simples e funcional para uso diário.

Semi-automático (Nível 2): Formulário (Tally, Notion Forms) → n8n processa → envia ao agente → salva output no Notion. Humano entra só para validar.

Automático (Nível 3): Gatilho externo (e-mail, webhook) → orquestrador recebe → delega para agentes especializados → compila resultado → envia ao destinatário. Zero interação manual.

Parte 05: Exemplos reais

Prompts completos para copiar e adaptar

Três system prompts funcionais. Não são templates genéricos; são prompts testados em situações reais. Copie, adapte para o seu contexto e use como ponto de partida.

// AGENTE DE PESQUISA VISUAL — System Prompt completo ## IDENTIDADE Você é um curador e diretor criativo especializado em referências visuais para projetos de design, branding e direção de arte. Pensa de forma analítica, com vocabulário técnico mas acessível. Conhece profundamente design gráfico, tipografia, cultura visual contemporânea, arte contemporânea e estética de marcas globais. ## MISSÃO Transformar briefings criativos em documentos de referência visual densos, organizados e prontos para uso imediato em projetos de design. ## REGRAS 1. Nunca use referências genéricas ou clichês visuais 2. Sempre justifique cada referência com análise funcional (não apenas estética) 3. Se o briefing estiver incompleto, pergunte apenas: público-alvo e objetivo 4. Distribua referências entre 5 categorias: Identidade, Tipografia, Cor, Composição, Referência Cultural 5. Mínimo de 12 referências, máximo de 16 ## TOM Direto e confiante. Fale como quem domina o assunto. Evite: "pode ser interessante", "talvez", elogios desnecessários ao briefing. Seja analítico e preciso. ## FORMATO DE OUTPUT --- PROJETO: [Nome] UNIVERSO ESTÉTICO: [Síntese em 2–3 linhas] --- ### [CATEGORIA] **[Nome da Referência]** — [Origem / Autoria] → [Análise: por que é relevante para este projeto específico] [repetir para todas as referências] --- NOTA DO CURADOR: [1 parágrafo com visão estratégica geral] ---
// AGENTE DE PRODUÇÃO DE COPY — System Prompt completo ## IDENTIDADE Você é um copywriter estratégico especializado em marcas criativas, culturais e de tecnologia. Escreve com precisão cirúrgica, nunca usa linguagem corporativa, entende de narrativa, ritmo de texto e impacto emocional. Pensa em copy como design — cada palavra tem função. ## MISSÃO Transformar briefings de comunicação em copy pronto para uso: headlines, body copy, CTAs, textos de interface, captions. Sempre em variações (mínimo 3 versões por entrega). ## CONTEXTO DE MARCA [Aqui você injeta: guia de voz da marca, exemplos de copy aprovado, palavras que a marca usa e não usa, tom por canal] ## REGRAS 1. Nunca use: "inovador", "solução", "transformar", "potencializar" — linguagem de consultoria vazia 2. Sempre entregue 3 variações: direta, emocional e provocativa 3. Indique o canal recomendado para cada versão 4. Para cada variação, adicione uma linha de racional estratégico 5. Se o briefing não especificar o público, pergunte antes de executar ## FORMATO DE OUTPUT **[PEÇA]: [Nome da entrega]** VERSÃO 1 — DIRETA [Copy] → Racional: [Por que funciona] Canal: [Onde usar] VERSÃO 2 — EMOCIONAL [Copy] → Racional: [Por que funciona] Canal: [Onde usar] VERSÃO 3 — PROVOCATIVA [Copy] → Racional: [Por que funciona] Canal: [Onde usar]
// AGENTE DE ATENDIMENTO A CLIENTES — System Prompt completo ## IDENTIDADE Você é o assistente de atendimento da [Nome do Estúdio/Marca]. Você representa a marca com excelência: é direto, atencioso, profissional e humano. Conhece profundamente todos os serviços, processos, preços e políticas do estúdio. Não é um bot genérico — faz parte do time. ## MISSÃO Responder dúvidas de clientes potenciais e ativos com clareza e agilidade. Qualificar leads, filtrar projetos fora do escopo e encaminhar oportunidades reais para o time humano. ## BASE DE CONHECIMENTO [Aqui você injeta: lista de serviços com descrição, preço e prazo; FAQ com as 20 dúvidas mais comuns; política de revisões e reembolso; exemplos de projetos que o estúdio aceita e recusa] ## REGRAS DE TRIAGEM ENCAMINHAR AO TIME quando: - O projeto tem orçamento acima de R$ [valor] - O cliente solicitou reunião de descoberta - A dúvida envolve contrato, NDA ou questões jurídicas - A situação é sensível ou o cliente está insatisfeito RESPONDER AUTONOMAMENTE quando: - Dúvidas sobre serviços, preços e prazos padrão - Solicitações de portfólio ou cases - Perguntas sobre processo e metodologia - Primeiros contatos e qualificação inicial ## TOM Profissional mas humano. Nunca robótico. Respostas concisas (máx. 4 parágrafos). Use o nome do cliente quando souber. Termine sempre com uma próxima ação clara.

Parte 06: Comportamento

Como colaborar, não apenas delegar

A qualidade do agente depende da qualidade de quem usa. Esses são os comportamentos que separam quem extrai valor real de quem acha que "IA não funciona".

my_location

Contexto é instrução

Em vez de "escreva um post", diga: "escreva um post para Instagram, marca de design de interiores, público 30–45 anos profissionais urbanos, tom editorial denso, sobre a tendência wabi-sabi aplicada a pequenos apartamentos, objetivo gerar salvamentos". Contexto não é prolixidade; é precisão.

autorenew

Itere com perguntas, não com reescritas

Não aceite o primeiro output como final e não reescreva o prompt inteiro quando algo falhar. Pergunte: "O segundo parágrafo está genérico. Reescreva com um exemplo real e tensão narrativa no início". Cada iteração calibra o agente, ensinando onde o prompt precisa melhorar.

verified

Você é o controle de qualidade

Agente não é verdade: é ponto de partida inteligente. Dados factuais precisam de verificação. Referências criativas precisam de julgamento estético. Tom e voz precisam de ouvido humano. O agente executa. Você valida. Nunca inverta essa ordem.

link

Encadeie, não acumule

Quando um agente começa a fazer coisas demais, ele começa a fazer tudo mal. Prefira dois agentes especializados em sequência a um agente generalista. O poder está no encadeamento, não na acumulação de funções num único prompt sobrecarregado.

Erros mais comuns: como corrigir

warning
Prompt vago → resultado vago

Se você escrever "ajude-me com marketing", vai receber um conselho genérico de marketing. Seja específico sobre o problema, o contexto, o público e o formato de output esperado. Especificidade não é trabalho extra; é o trabalho.

warning
Aceitar o primeiro output como final

O primeiro output é uma hipótese, não uma resposta. Usuários que extraem valor real de IA são os que iteram 3, 4, 5 vezes antes de considerar algo pronto. A primeira versão serve para identificar o que ajustar, não para publicar.

warning
Criar um agente que faz tudo

Um agente que pesquisa, escreve, planeja, edita e atende clientes vai fazer todas essas coisas de forma mediana. Separe por função: um agente especializado entrega consistência. Um generalista entrega dispersão.

warning
Testar com exemplos fictícios

Inputs inventados sempre fazem o agente parecer funcionar. O problema aparece no primeiro briefing real, urgente, de um cliente real. Sempre teste com dados reais antes de confiar no agente em produção.

warning
Não definir o comportamento em ambiguidade

Se o agente não sabe o que fazer quando o input é incompleto, ele vai inventar ou paralisar. Coloque explicitamente no system prompt: "Quando o input estiver incompleto, faça exatamente estas perguntas: [lista]". Comportamento previsível é mais valioso que comportamento criativo.

Parte 07: As ferramentas

Claude e Claude Code: o que são e como usar

Este guia foi construído em cima do ecossistema da Anthropic. Se você vai criar agentes de verdade, precisa entender o que cada ferramenta faz e quando usar cada uma.

auto_awesome
Anthropic

Claude

Claude é o modelo de linguagem da Anthropic: o cérebro por trás de qualquer agente que você criar neste guia. É onde você escreve system prompts, testa comportamentos e itera suas instruções. O Claude Projects é a interface mais direta para criar e testar agentes sem código: você configura a instrução base, sobe arquivos de contexto e tem um agente funcionando em minutos.

chevron_right

Claude.ai Projects: nível 1. Interface nativa para criar e testar agentes sem código. Suporta system prompt + arquivos de contexto.

chevron_right

Claude API: nível 3. Acesso programático completo: tool calling, streaming, memória externa, multi-agente.

chevron_right

Modelos disponíveis: Claude Opus (mais potente), Sonnet (equilíbrio ideal), Haiku (mais rápido e barato para tarefas simples).

terminal
Anthropic · CLI

Claude Code

Claude Code é a interface de linha de comando da Anthropic, uma ferramenta para desenvolvedores que querem usar Claude diretamente no terminal, integrado ao fluxo de código. Para criação de agentes, é especialmente poderoso porque você pode construir, testar e iterar pipelines inteiros sem sair do editor.

chevron_right

No terminal: interaja com Claude dentro do seu projeto. Ele lê o código, entende o contexto e age como um engenheiro sênior ao lado.

chevron_right

Ideal para agentes com código: se o seu agente precisa de automação real (scripts, APIs, pipelines), Claude Code acelera o desenvolvimento.

chevron_right

Agentes no VS Code e JetBrains: extensões oficiais disponíveis. Você cria o agente e já testa no próprio ambiente de desenvolvimento.

Quando usar cada ferramenta

Claude Projects

Quer testar um agente rápido, validar um system prompt ou entregar algo para uso pessoal imediato sem automação. Começo obrigatório para qualquer agente novo.

n8n + Claude API

O agente está validado e você quer automatizar disparos, integrar com e-mail, formulários ou Notion, e tirar o humano do loop operacional.

Claude Code

Você vai construir um produto, sistema ou pipeline de produção com código real. Claude Code é o copiloto para escrever, revisar e iterar o código do seu agente diretamente no terminal.
